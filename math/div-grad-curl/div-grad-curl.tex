\section{Real-Valued Derivative, Partials, and Gradient}

\begin{enumerate}

\item With a function of a single variable, we have already defined the
\emph{derivative}:

\begin{nedqn}
  \fderiv{x} f(x)
\eqcol
  \lim_{a \to x}
  \frac{
    f(a) - f(x)
  }{
    a - x
  }
\\\eqcol
  \lim_{t \to 0}
  \frac{
    f(x + t) - f(x)
  }{
    t
  }
\end{nedqn}

\item The notation $\fderiv{x} f(x)$ is called \define{Leibniz
notation}. There is also \define{Lagrange notation}, in which we write
$f'(x)$, which is common in univariate settings, but it will be
difficult to know what is being differentiated in multivariate settings.
There is \define{Newton notation}, where we write $\dot f$; this is used
in physics sometimes, the presumption is always that we are
differentiating against time. Last, there is \define{Euler notation},
where we write $\opD_x f(x)$.

\item Let's begin to generalize the univariate derivative to a function
of many variables. For now, we'll keep $f$ mapping to a single real
variable. Then we define the \define{partial derivative}:

\begin{nedqn}
  \fpartial{x_i} f(x_1, \ldots, x_n)
\eqcol
  \lim_{t \to 0}
  \frac{
    f(x_1, \ldots, x_i + t, \ldots, x_n)
    -
    f(x_1, \ldots, x_n)
  }{
    t
  }
\end{nedqn}

\item A very typical case is going to be when, instead of treating
separate parameters, we can consider the parameters to be packaged as a
``vector'', and the domain of the function is a normed vector space. A
very typical case will be $f: \R^n \to \R$. In that case:

\begin{nedqn}
  \fpartial{x_i} f(\vx)
\eqcol
  \lim_{a \to \vx_i}
  \frac{
    \ff{\tilde\vx} - \ff{\vx}
  }{
    \norm{
      \tilde\vx - \vx_i
    }
  }
\intertext{where}
  \tilde{\vx}
\eqcol
  \parens{\vx_1, \ldots,a, \ldots, \vx_n}
\intertext{we can also write}
  \fpartial{x_i} f(\vx)
\eqcol
  \left.
    \fderiv{t} f(\vx + t\ve_i)
  \right|_{t=0}
\intertext{where $\ve_i$ is the $i$-th basis vector.}
\end{nedqn}

\item The partial derivative effectively restricts $f$ to a function of
just $x_i$, where all other parameter values $x_j$ are held constant as
$\vx_j$. Then, we calculate the typical univariate derivative at $x_i$.
This partial derivative is basically the slope of the function in the
direction of $x_i$.

\item Sometimes people write $f_x$ or $f'_x$ for the partial derivative.
The first looks like a restriction of the function $f$ to the
$x$-variable, instead of a derivative; I wouldn't use that. $f'_x(x, y)$
is similar to the Lagrange notation, but I like the Leibniz notation
better ($\fpartial{x}$). The Euler notation $\opD_x$ is also good.

\item We typically say ``partial derivative'' when we are talking about
a derivative with respect to either an individual ``parameter'' or a
``coordinate'' basis vector. But that might be a bit arbitrary of us. If
$f$ maps from a vector space, we can take a derivative along \emph{any}
direction in the space. We call this a \define{directional derivative}:

\begin{nedqn}
  \opD_{\vv} f(\vx)
\eqcol
  \lim_{t\to 0}
    \frac{
      f(\vx + t\vv) - f(\vx)
    }{
      t
    }
\intertext{equivalently}
\eqcol
  \left.
    \fpartial{t} f(\vx + t\vv)
  \right|_{t=0}
\end{nedqn}

\item Wikipedia at least writes $\grad_{\vv} f(\vx)$, but both that
notation (and $\opD_{\vv}$, tbh) are a bit odd because they seem to
suggest the existence/use of \emph{other} derivatives. The notation
$\fpartial{\vv} f(x)$ attempts to generalize the partial derivative
notation, but ChatGPT suggests this may not be a standard notation. I do
think maybe the $\opD_{\vv}$ notation is probably the most preferred.
One downside is that, if $f$ is not (total) differentiable at $\vx$, use
of $\opD_{\vv} f(\vx)$ might be misleading if it suggests the total
derivative $\opD f(\vx)$ exists.

\item A function $g$ is a \define{first-order approximation} of $f$ if
at $\vx$ if, for any direction $\vu$:

\begin{nedqn}
  \lim_{t \to 0}
  \frac{
    g(\vx + t\vu) - f(\vx + t\vu)
  }{
    t \norm{\vu}
  }
\eqcol
  0
\end{nedqn}

\item The function $g(\vu) = f(\vx) + \opD_{\vu} f(\vx)$ is a
first-order approximation of $f$ at $\vx$, almost by definition. Of
course, this first-order approximation only exists if $\opD_{\vu}$
exists.

However, if $\opD_{\vu} f(\vx)$ is too ``weird'' (directional derivative
changes too wildly as direction changes), we will not call $f$
\define{(total) differentiable} at $\vx$. For $f$ to be considered
(total) differentiable at $\vx$, we will require that there exists an
\define{affine} first-order approximation. This occurs exactly when
$\opD_{\vu} f(\vx)$, when treated as a function of $\vu$, is
\emph{linear}. At this point, we define $\opD f(\vx)$ to be the linear
functional:

\begin{nedqn}
  \left(\opD f (\vx)\right) \vu
\defeq
  \opD_{\vu} f(\vx)
\end{nedqn}

\noindent
We call $\opD f(\vx)$ the \define{(total) derivative} of $f$ at $\vx$.
It is like the ``slope'' of the (multivariate) affine first-order
approximation for $f$ at $\vx$.

\item In the univariate case, there was no need to separate the ideas of
partial/directional derivative and total derivative. In the multivariate
case, the existence of a directional derivative along every direction
does not imply that a \emph{linear} total derivative exists. The
linearity of directional derivatives is exactly what is needed
for a first-order approximation of $f$ at $\vx$ to exist.

\item We can define the \define{gradient}. It is a vector, and to write
it down, we need to have fixed a coordinate basis. The gradient is
defined:

\begin{nedqn}
  \gradient f(\vx)
\eqcol
  \begin{pmatrix}
    \fpartial{x_1} \ff{\vx}
    \\
    \vdots
    \\
    \fpartial{x_n} \ff{\vx}
  \end{pmatrix}
\end{nedqn}

\noindent
The gradient is useful because (when $f$ is (total) differentiable):

\begin{nedqn}
  \left(\opD f(\vx) \right) \vu
=
  \opD_{\vu} f(\vx)
=
  \gradient f(\vx) \cdot \vu
\end{nedqn}

\noindent
Provided the function is relatively well-behaved, the gradient allows us
to calculate the directional derivative in any direction. Because $\opD
f(\vx)$ is required to be linear, its value at $n$ basis vectors is
enough to specify it.

\item Note that our gradient has the shape of a ``column'' vector in the
space, so the gradient is often interpreted as a \emph{direction}
(``directly uphill''). On the other hand, $\opD f(\vx)$ is a linear
functional (a map from vectors to scalars), and so can be interpreted as
a \emph{co-vector} (a ``row'' vector). In a sense, $\opD$ and
$\gradient$ are the same, just \emph{transposes} of each other.

\section{Vector Valued Functions}

\begin{enumerate}

\item The other direction of generalization is to consider
\define{vector-valued functions}. A vector-valued function $\vec{f}:
\R^m \to \R^n$ is also sometimes called a \define{vector field}. A
vector space is just a \emph{space} of vectors, like $\R^n$. ``Field''
here is at odds with the definition in \emph{algebra} (where the
rational numbers, $\mathbb{Q}$, is an example of an algebraic field).
The use of vector field as an assignment of a vector to each point in a
vector space is really more of a physics and vector calculus name.

\item Famous examples of vector fields are electric fields, magnetic
fields, gravitational fields, fluid velocity fields.

\item Our prior definition of a partial derivative (and of a directional
derivative) will still apply. We just need to note that the numerator in
those definition is a vector. We'll need a normed vector space.

We will need the vector space to be \emph{normed}, because we need a
notion of convergence so that a limit can be defined.

\item Instead of generalizing the gradient, we'll generalize the total
derivative. That was equivalent to a co-vector (a matrix of one row, the
transpose of the gradient vector). The vector-valued total derivative
will correspond to a \emph{matrix} that we call the \define{Jacobian}:

\begin{nedqn}
  \mJ_{\vec{f}}(\vx)
\eqcol
  \begin{bmatrix}
    \grad\tran f(\vx)_1 \\
    \grad\tran f(\vx)_2 \\
    \vdots \\
    \grad\tran f(\vx)_n \\
  \end{bmatrix}
\end{nedqn}

\noindent
Here, I $f(\vx)_1$ just means the restriction of the value of $f$ to the
first-coordinate. $\gradient\tran f(\vx)_1$ is effectively the
representation of the linear functional $\opD_{f_1}(\vx)$.

\item As before, provided the function really is differentiable and thus
the total derivative exists, then:

\begin{nedqn}
  (\opD \vec{f} (\vx)) \vv
\eqcol
  \mJ(\vx) \vv
\end{nedqn}

\noindent
Note that the Jacobian is more like a representation the total
derivative than the gradient is, in the sense that you don't have to
transpose it.

\end{enumerate}

\section{Line and Surface Integrals}

\subsection{Line Integral}

\begin{enumerate}

\item We can next define a \define{line integral} (also called a path
integral). First we have to define a \define{curve}. A curve $\calC$ is
a subset of a vector space (usually $\R^n$) with the following
requirement: there is a continuous map $r: [a, b] \to \R^n$, and the
image of $r$ is $\calC$ ($[a, b]$ is a closed interval in $\R$, of
course). Everything probably remains fine whether the curve is closed or
open.

\item Now we can define the line integral of $f$ along a curve $\calC$
defined by $r$. We'll want $r$ to be piecewise differentiable. It is
defined to be:

\begin{nedqn}
  \int_\calC
  f \diff{s}
\eqcol
  \int_a^b
  \ff{\vr(t)}
  \norm{\vr'(t)}
  \dt
\end{nedqn}

\noindent
Here the notation $\vr(t)$ emphasizes that this is a vector-valued
function of $t$. Note that the ``rate'' at which we move through the
curve (which is arbitrary) is corrected by $\norm{\vr'(t)}$. This seems
to make the line integral independent on the parameterization of
$\calC$, but in fact any parameterization will give the same integral
result (would need proof!).

\item We define the \define{length} of a curve as $\int_\calC 1
\diff{s}$.

\item For now, I will consider $\int_\calC f \diff{s}$ as pure
notation. It feels pretty meaningless and arbitrary at present.

\item Note that the path-integral concept trivially generalizes to a
vector-valued function $\vec{f}$.

\item It feels a little bit like length should be a property of the
curve in the vector space itself and a definition of unit length for
unit vectors. It feels like a bit of an intrusion that we would need to
introduce to a parameterization like $\vr$, and use it so centrally in
the definition of the length of a curve.

\item But I believe differentiability of the curve will always be
necessary because length of the curve has to do with a integral of the
magnitude of the tangency vector. And you will want to control the rate
at which you trace the curve, so that you don't ``cheat'' by moving
through different parts at different rates.

I can't see a cleaner, simpler way to do this than by introducing $\vr$.
Whatever we do probably will always end up equivalent to introducing
$\vr$.

\end{enumerate}

\subsection{Surface Integral}

\begin{enumerate}

\item We can define a corresponding \define{surface integral}. We must
again first define what we mean by a \define{surface}. A surface $S$ is
a subset of a vector space (typically $R^n$) with the following
property: there exists a continuous mapping $r: U \to \R^n$ ($U$ should
be an open set in $\R^2$) where $S$ is the image of $r$.

I think that it's fine whether the set is open or closed, so long as
it's not too misbehaved. It would probably be best if the
parameterization is bijective, but that probably isn't strictly
necessary.

\item We should next define the notion of \define{surface area} (just as
we defined curve length). My first attempt was to try define this as:

\begin{nedqn}
  \iint_S
  1 \diff{S}
& \stackrel{?}{=} &
  \iint_U
  \det \left(\mtxJ_{\vr}(\vu)\right)
  \diff{\vu}
\end{nedqn}

\noindent
Here we use that $\det \mtxJ_{\vr}(\vu)$ is the amount by which area is
stretched by $\vr$ in the vicinity of $\vu$. However, since $\det$ is
only defined for \emph{square} matrices, this won't work when we are
mapping from two-dimensions to three!

\item Really, we should use the cross-product:

\begin{nedqn}
  \iint_S
  1 \diff{S}
\eqcol
  \iint_T
  \norm{
    \frac{\partial \vr}{\partial x}(\vu)
    \cross
    \frac{\partial \vr}{\partial y}(\vu)
  }
  \diff{x} \diff{y}
\end{nedqn}

\noindent
This is correct, because $\va \cross \vb$ has magnitude equal to the
area spanned by the two vectors: $\va \cross \vb = \norm{\va}\norm{\vb}
\sin(\theta)$.

\item We can match this to my intuition to use the determinant. First,
note:

\begin{nedqn}
  \det A
\eqcol
  ad - bc
\intertext{and}
  \colvec{a \\ c \\ 0} \cross \colvec{b \\ d \\ 0}
\eqcol
  \parens{
    ad - bc
  } \vk
\end{nedqn}

\noindent
So while the correct definition is definitely to use the cross-product,
the determinant idea was on the right track.

\item I would note that area is defined by height times width, where
those are measured orthogonally. We're assuming that $x$ and $y$ are
orthogonal directions in the space. So the local stretching of a space
depends on (1) how much differences in $x$ and $y$ get stretched, and
(2) how much they remain orthogonal in the target space.

\end{enumerate}

\section{Divergence}

\begin{enumerate}

\item We can finally define the \define{divergence} of a vector field
$\mtxF$ at a point $\vu$:

\begin{nedqn}
  \operatorname{div} \mtxF(\vu)
\defeqcol
  \lim_{V \to 0}
  \frac{1}{\abs{V}}
  \iint_{S(V)}
  \mtxF(\vx) \cdot \vn(\vx)
  \diff{\vx}
\end{nedqn}

\noindent
What the hell does this mean? First, we will $V$ is a volume which
contains $\vu$. The limit is expressing that we take a series of volumes
where the volume of $V$ (denoted $|V|$) is approaching zero. For
instance, $V$ could be a series of $\epsilon$-balls, where $\epsilon \to
0$.

Next, we are going to do a surface integral. The surface we integrate
over will be the surface of the volume. This could, for instance, be the
$\epsilon$-shell.

What will we integrate? $\mtxF(\vx) \cdot \vn(\vx)$. What is $\vn(\vx)$?
It will be the unit-vector that is \define{normal} to the surface $S(V)$
at position $\vx$.

\item For a field $\mtxF$, we define the \define{flux} $\Phi_{\mtxF}(S)$
to because

\begin{nedqn}
  \Phi_{\mtxF}(S)
\defeqcol
  \iint_S
  \mtxF(\vu) \cdot \vn(\vu)
  \diff{\vu}
\end{nedqn}

\noindent
Thus, with this definition, we can equivalently define:

\begin{nedqn}
  \operatorname{div} \mtxF(\vu)
\defeqcol
  \lim_{\abs{V_i} \to 0}
  \frac{
    \Phi_{\mtxF}(S_i)
  }{
    \abs{V_i}
  }
\end{nedqn}

\item So the flux through a surface is something like the amount being
pushed out through the surface. If the vector field defines the velocity
of a fluid, then the flux is the instantaneous rate of fluid flow across
the surface.

\item The divergence is similar. It is the amount of flux \emph{per unit
area} across an infinitesimal ball centered at a point $\vx$.

\end{enumerate}
